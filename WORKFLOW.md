# 🌸 Contribution Workflow – CodeLens

Welcome to CodeLens! 💛  
We’re really happy to have you here and excited that you’re interested in contributing.

To keep things smooth, collaborative, and comfortable for everyone, please follow the workflow below.

---

## ✨ Getting Started

If you find an issue interesting, you can **start working on it directly**.  
There’s no strict need to wait for assignment.

If you’d like, you can leave a comment saying you’re working on it — this helps avoid overlap and keeps things transparent.

We truly appreciate your initiative and enthusiasm 🌱

---

## 🛠️ Working on the Issue

- Take your time to understand the problem
- Follow the existing project structure
- Keep your code clean and readable
- Focus on the scope of the issue

---

## 📸 Sharing Your Work (Important)

When you create a Pull Request, please include:

- A clear explanation of your changes  
- Screenshots or screen recordings (for UI updates)  
- Steps to test your implementation  

This helps us review your work faster and better ✨

---

## ❓ If You Need Help

If you face any issues or confusion:

- Feel free to ask in the issue comments  
- Or use GitHub Discussions  

No question is too small — we’re here to help you grow 💛

---

## 🔍 Before Creating a PR

Just quickly check:

- Your code is working properly  
- Changes are tested  
- You followed project structure  
- PR is clean and focused  

---

## 🤝 A Friendly Note

We believe in:
- Supporting each other  
- Learning together  
- Building with respect and kindness  

Your contributions truly matter here 🌸

---

## ⭐ Final Thought

Take initiative, explore freely, and enjoy the process.  
We’re excited to see what you build with CodeLens 🚀

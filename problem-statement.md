# Problem Statement: CodeLens

## Background
The rapid evolution of software engineering has led to an explosion of tools, standards, and architectures. However, developers often struggle to maintain a consistent growth trajectory and understand their core competencies objectively.

## The Problem
There is a lack of cohesive, AI-driven platforms that not only track developer activity but provide actionable, deeply personalized insights and roadmaps. Existing tools are highly fragmented: some focus on LeetCode/CP, others on GitHub activity, while a few offer isolated AI coaching. This fragmentation leads to:
1. Imposter syndrome or lack of clear direction.
2. Inefficient skill-building routines.
3. Poorly managed open source involvement.

## The Solution (CodeLens)
CodeLens unifies data from multiple sources (GitHub, Coding Platforms like LeetCode/Codeforces) and leverages Large Language Models (Gemini API) to dynamically analyze code, track growth, and provide clear, tailored steps to help the developer improve systematically over time. It is a holistic AI mentor and tracking panel specifically designed to combat developer stagnation.
